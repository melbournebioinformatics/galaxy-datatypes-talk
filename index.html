Defining a custom datatype

  • Simplest to just subclass an existing datatype
  • 
<datatype extension="special_tabular" 
type="galaxy.datatypes.tabular:Tabular" subclass="True"/>
  • Useful if special_tabular is produced and consumed by specific tools
  • Define in datatypes_conf.xml within a toolshed tool or "hack" into Galaxy's datatypes_conf.xml

More advanced datatype behaviour requires a custom Python class

Composite datatypes

  • Often used for data that requires companion files (eg indexes)
  • Two types, basic and auto_primary_file
  • Not well supported by the new file uploader
  • The Wiff datatype demonstrates auto_primary_file
